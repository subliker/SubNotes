# SubNotes — план проекта

## Что это

Лёгкий **компаньон поверх системного Календаря macOS**. Не дублирует календарь:
Google-события уже синкаются в Calendar.app через «Учётные записи интернета», а
SubNotes добавляет сверху то, чего в стандартном Календаре нет.

Три поверхности ценности:
1. **Список событий в поповере из menu bar** — клик по иконке открывает список
   ближайших событий (как виджет на Android, но из строки меню). Заменяет
   WidgetKit-виджет, чтобы уйти от подписи/App Group.
2. **Бегущая строка в menu bar** (а-ля RunCat) — напоминания умным появлением.
3. **Интерактивные оверлеи** поверх экрана — кастомизируемые скины (от карточки
   до «девочки-аниме с плакатом» или пролетающего самолёта с баннером).

Собственное окно приложения — **только настройки**.

> **Почему не WidgetKit-виджет:** настоящий desktop-виджет требует App Group для
> обмена данными с отдельным sandboxed-процессом, а App Group на macOS требует
> Team ID и provisioning — то есть какой-либо учётки разработчика. Это
> несовместимо с «полностью неподписанным» (проверено в Фазе 0). Поэтому список
> событий живёт в поповере главного процесса — ноль подписи, работает у всех.

## Ключевые решения

| Тема | Решение |
|---|---|
| Платформа | Нативно: **Swift + SwiftUI + AppKit** |
| Данные календаря | **EventKit** (чтение системных календарей). Google уже подключён через Internet Accounts — OAuth не нужен. |
| Режим данных | **Read-only** в MVP. Edit — возможно позже; границу read/write держим чистой, но edit-кода сейчас НЕ пишем. |
| Присутствие | **Только menu bar (LSUIElement)**, без Dock-иконки. |
| Окно приложения | Только экран **настроек**. |
| Открытие события | Deep-link в системный **Calendar.app**. |
| Напоминания | По **EKAlarm** событий. |
| Стиль | **Liquid Glass** → целимся в свежую macOS (26 Tahoe+), без back-compat компромиссов. |
| Распространение | **Open source, неподписанный `.dmg`**. Никаких платных Apple-аккаунтов. Раздача через GitHub Releases. |

## Архитектура

```
SubNotes/
├── Sources/App/          # menu-bar приложение (LSUIElement), один процесс
│   ├── SubNotesApp       # MenuBarExtra + поповер со списком событий
│   ├── TickerStatusItem  # NSStatusItem с бегущим текстом (RunCat-style)  [Фаза 3]
│   ├── OverlayEngine     # прозрачное NSWindow + движок скинов            [Фаза 4]
│   └── Settings          # окно настроек (SwiftUI)                        [Фаза 5]
└── Packages/CalendarCore # Swift package: модели + чтение EventKit + парсинг видеоссылок
```

**Поток данных:** EventKit → CalendarCore (нормализует) → прямо в UI того же
процесса (поповер, ticker, оверлеи). App Group/снапшот больше не нужны — всё в
одном процессе.

## Поверхности (спецификация MVP)

### Список событий в поповере (menu bar)
- Клик по иконке → окно-поповер (`.menuBarExtraStyle(.window)`) со списком
  ближайших событий: цвет календаря, название, время.
- Клик по событию → открыть в Calendar.app.

### Бегущая строка (menu bar, RunCat-style)
- `NSStatusItem` с прокручивающимся текстом ближайших событий.
- **Умное появление**: показывается только за N минут до события (дефолт 15), потом прячется.
- Клик → открыть событие в Calendar.app. Меню: «что дальше» + toggle'ы фишек + Настройки.

### Оверлеи со скинами
- Прозрачное полноэкранное `NSWindow`, click-through в пустых зонах.
- Триггер по EKAlarm.
- **Движок тем**: рендер по манифесту скина (JSON: ассеты, зоны текста, анимация, длительность, звук, расположение кнопок).
- **Формат темы** `.subnotes-theme` (папка + manifest.json). **Встроенные + пользовательские** темы (кладёшь папку, без пересборки).
- MVP: **2 встроенных скина** — минималистичная карточка + анимированный «самолёт» (SpriteKit) как демо.
- Кнопки плавающим слоем поверх любого скина: **Отложить / Закрыть / Открыть / Подключиться** (Meet/Zoom — парсим из события).

### Настройки
Какие календари участвуют, горизонт дней, N минут для ticker, snooze-интервалы,
автозапуск, тема/скин.

## Порядок реализации

- **Фаза 0 — Спайк (де-риск). ✅ ВЫПОЛНЕНО.** Результат: настоящий WidgetKit-виджет несовместим с неподписанной сборкой (App Group требует Team ID/provisioning). Виджет заменён на поповер-список в menu bar. Каркас собран: Swift-пакет CalendarCore (модель `CalEvent`, `EventReader` на EventKit), menu-bar приложение с поповером, ad-hoc сборка `Signature=adhoc / TeamIdentifier=not set` без provisioning. Остался UI-прогон пользователем.
- **Фаза 1 — CalendarCore.** Запрос доступа, чтение событий на горизонт, свои модели, подписка на `EKEventStoreChanged`, парсинг видеоссылок. Чистая граница для будущего edit.
- **Фаза 2 — Поповер-список (MVP-цель).** Полноценный список (группировка по дням, цвета календарей, all-day), открытие в Calendar.app.
- **Фаза 3 — Ticker в menu bar.** NSStatusItem, бегущий текст, умное появление.
- **Фаза 4 — Оверлеи + движок скинов.** Прозрачное окно, манифест тем, 2 встроенных скина, пользовательские темы, очередь при одновременных напоминаниях, планировщик.
- **Фаза 5 — Настройки, login item, упаковка.** SMAppService автозапуск, `.dmg`, GitHub Actions, README (снятие карантина `xattr -dr com.apple.quarantine` + документация формата тем).

## 🟡 Открытые вопросы — продумать и при необходимости поменять

Зафиксированы дефолты, но это НЕ финальные решения:

**Данные**
- Горизонт загрузки — дефолт 7 дней вперёд. ⚠️ продумать.
- Повторяющиеся / all-day события — как показываем (all-day плашкой сверху дня). ⚠️
- Несколько аккаунтов/календарей — по умолчанию все видимые. ⚠️
- Таймзоны. ⚠️

**Ticker**
- Дефолт lead-времени появления — 15 мин. ⚠️
- Поведение на нескольких мониторах (для menu bar — экран с активным меню). ⚠️
- Что крутится: только ближайшее или все оставшиеся за день. ⚠️
- Ограниченная ширина элемента в menu bar — проверить читаемость. ⚠️

**Оверлеи**
- Позиция: дефолт правый верхний угол (настраиваемо). ⚠️
- Авто-исчезновение через X сек vs висят до реакции — дефолт «висят». ⚠️
- Дефолтные snooze-интервалы 5/10/15. ⚠️
- Несколько напоминаний одновременно — очередь. ⚠️
- Звук вкл/выкл по умолчанию. ⚠️
- Click-through + кликабельные кнопки на анимированном скине — технический риск. ⚠️

**Сквозное**
- Первый запуск / онбординг — поток запроса EventKit, поведение при отказе. ⚠️
- Системные `UserNotifications` vs наши оверлеи — дефолт только свои (избежать дублей). ⚠️
- Точность срабатывания: планировщик таймеров; «пропущенные» напоминания после сна/выключения Mac. ⚠️
- Хранилище в App Group: JSON-снапшот (дефолт для MVP) vs CoreData. ⚠️
- Язык интерфейса: RU / EN / оба. ⚠️
- Лицензия open source: MIT / GPL / Apache. ⚠️
- Иконка / брендинг SubNotes. ⚠️

## Риски

1. **App Group без подписи** — главный, проверяем в Фазе 0.
2. **Оверлеи поверх fullscreen** — нужен правильный `collectionBehavior`, проверить эмпирически.
3. **Liquid Glass API** — относительно новые материалы, проверить доступность/поведение.
4. **Click-through на анимированных скинах** — кнопки должны оставаться кликабельными поверх прозрачных зон.
